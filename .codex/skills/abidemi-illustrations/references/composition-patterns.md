# Composition Patterns

These are **starting points**, not templates. Invent fresh metaphors each time.

## Pattern 1: Before / After

Split screen (left = problem, right = solution). ABIDEMI on the right side, often holding the solution object.

**Example:** Left: phone with "No Quick Share" crossed out. Right: same phone with "Bada" badge. ABIDEMI standing next to right phone, pointing.

## Pattern 2: Obstacle + Key

A large block (the problem) dominates the center. ABIDEMI holds a small object (the solution) that unlocks or removes the block.

**Example:** Heavy "Chinese ROM" boulder. ABIDEMI holds a "Bada" key that opens a gate on the boulder.

## Pattern 3: Workflow Arrow

Horizontal sequence of 2–4 steps. ABIDEMI at one or more steps, performing the action.

**Example:** "Download APK" → "Install" → "Grant Permissions" → "Share". ABIDEMI at step 2, pressing an install button.

## Pattern 4: Ecosystem Map

Simple 2–3 node diagram (boxes with labels). ABIDEMI as the connector drawing lines or carrying objects between boxes.

**Example:** Box "Chinese Phone" connected to box "Mac/Windows" via a line labeled "Bada". ABIDEMI sitting on the line.

## Pattern 5: Metaphor Object

One central object that represents the abstract concept. ABIDEMI interacts with it in a surprising but logical way.

**Example:** "Quick Share" as a wall socket. ABIDEMI holding a "Bada" plug that fits perfectly.

## Rules for All Patterns

- Never use more than 3 objects + ABIDEMI.
- Annotate sparingly – one or two short words per object.
- If the concept needs more than 5 annotations, split into two images.
- For MP4/Canvas outputs, animate only ONE element at a time. Do not animate everything simultaneously.
