## Usage Instructions

### Referencing During Phase 3 Recommendations
After recommending a design direction, show the corresponding pre-made screenshot:
```
"This is what the Pentagram style looks like for a WeChat cover -> [show cover/cover-pentagram.png]"
"This is the feel of the Takram style for a PPT data page -> [show ppt/ppt-takram.png]"
```

### Scenario Matching Priority
1. User's request matches exactly -> show the matching scenario directly
2. No exact match but similar type -> show the closest scenario (e.g., "product website" -> show SaaS landing page)
3. No match at all -> skip pre-made samples, go directly to Phase 3.5 live generation

### Side-by-Side Comparison Display
3 styles for the same scenario work well displayed side by side for intuitive comparison:
- "Here's the same WeChat cover implemented in 3 different styles"
- Display order: Pentagram (Rational & Restrained) -> Build (Luxurious Minimalism) -> Takram (Soft & Warm)
